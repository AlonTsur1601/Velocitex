## How Codex and GPT-5.6 were used

Codex was used throughout Velocitex’s implementation and verification workflow. It helped turn design decisions into Godot/C# changes, build and refine momentum mechanics and room systems, investigate physics and route failures, and create repeatable PowerShell build, export, and smoke-test scripts. GPT-5.6, used through Codex, materially supported the reasoning and iteration behind those tasks. The creator made the final product, design, and engineering decisions. The repository README and evidence files document the workflow and the Build Week additions.

## Testing instructions

1. Download `Velocitex.zip` from the published GitHub Release.
2. Extract the entire archive to a writable local folder.
3. Run `Velocitex.exe` without moving it away from `Velocitex.pck` or `data_Velocitex_windows_x86_64`.
4. Start a new campaign and complete Room 01 by following the slope and acceleration rings to the collection cup.
5. Press Esc to test the pause menu and room restart. From the main menu, inspect Customize, Advancements, and Settings.

Controls: WASD or Arrow Keys to move while grounded; Mouse to look; C to switch camera; Middle Mouse to zoom; E to interact; Esc for pause menu.
